# Update Context Files

## Objective
Update project context files based on new information or changing requirements.

## Usage
```
/update-context
```

## Context Files to Review

### 1. Company Profile (`context/company-profile.md`)
Review and update:
- Company capabilities and core competencies
- Strategic goals and priorities
- Constraints and limitations
- Available resources (team, budget, infrastructure)
- Recent developments or changes

### 2. Client Information (`context/client-info.md`)
Review and update:
- Client background and industry position
- Client goals and success criteria
- Client constraints (budget, timeline, regulatory)
- Stakeholder preferences
- Communication preferences

### 3. Industry Background (`context/industry-background.md`)
Review and update:
- Industry trends and dynamics
- Regulatory landscape changes
- Competitive environment shifts
- Technology advancements
- Market conditions

## Update Process

### 1. Review Existing Context
- Read current context files
- Identify outdated information
- Note gaps in coverage

### 2. Gather New Information
- Research recent developments
- Interview stakeholders (if applicable)
- Review internal documents
- Monitor industry news

### 3. Update Files
- Revise outdated sections
- Add new relevant information
- Remove obsolete content
- Maintain consistent format

### 4. Validate Impact
- Identify active sprints that may be affected
- Consider re-running analysis if context changes significantly
- Document rationale for major changes

## Output
- Updated context files
- Change log documenting updates
- Impact assessment on active/completed sprints
